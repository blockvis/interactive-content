# Slide 10 — Долг №3: $\mathbb Z[i]$ ↔ $\mathbb Z[\omega]$

> Читать сначала: [`_shared.md`](./_shared.md), затем `../BRIEF.ru.md` §§ 4 (таблица долгов), 7 (строка слайда 10), §14 (цитата про решётку).

## Роль в арке доклада

Третий и главный «технический долг». Обычная картина «целых чисел на
плоскости» — это $\mathbb Z[i]$, гауссовы целые: квадратная решётка.
У каждой точки **4 ближайших соседа**. Но эта решётка не плотнейшая в
плоскости; плотнейшая — **гексагональная**, $\mathbb Z[\omega]$, где
$\omega = e^{2\pi i/3}$. У её точек **6 ближайших соседей**.

Авторская формулировка из BRIEF §14:

> *«The hexagonal lattice is not a generalisation of the Gaussian
> square lattice — it is a correction of it.»*

Слайд готовит слайды 11–12 (цифры Госпера и остров Госпера), где уже
не «теоретически», а **практически** покажется, как эта поправка
работает в арифметике.

Биологический мост (см. backstage): **fovea** человеческого глаза —
это уже $A_2$-решётка, природа выбрала гексагональную упаковку за нас.

## Длительность и «кончик скотча»

- На сцене: 60–75 секунд.
- **Кончик скотча:** квадратная решётка на плоскости, одна точка в
  центре, у неё подсвечены 4 ближайших соседа. Жест: **переключить
  в гексагональную** — и вдруг соседей шесть, плотнее упаковка,
  круги (шары) касаются друг друга по-другому.

## Content

### Визуал

Сцена — одна плоскость (SVG ~700×500 px).

В центре — выделенная «центральная» точка (жирный кружок, цвет
`var(--class-accent)`). Вокруг неё — решётка точек.

В **каждой** точке нарисован маленький круг (radius = half-lattice-step),
символизирующий **шар** на своей позиции — то есть решётка показана
как **упаковка кругов**, а не просто как набор точек.

Над сценой — переключатель **«$\mathbb Z[i]$ ↔ $\mathbb Z[\omega]$»**.

- В режиме $\mathbb Z[i]$ (квадратная):
  - Решётка квадратная.
  - Радиус кругов — половина шага решётки. Круги **касаются** друг
    друга по осям, но оставляют **пустоты** в углах.
  - 4 ближайших соседа центральной точки подсвечены
    (цвет `var(--class-accent)`), между ними и центром нарисованы
    тонкие пунктирные линии.
  - Счётчик «соседей: **4**» в углу сцены.
- В режиме $\mathbb Z[\omega]$ (гексагональная):
  - Решётка гексагональная. Базисные векторы: $1$ и $\omega = e^{2\pi
    i / 3}$.
  - Радиус кругов — половина шага решётки (одинаковый шаг = расстояние
    между ближайшими соседями).
  - **Круги касаются со всех сторон**, без пустот: это и есть
    плотнейшая упаковка в 2D.
  - 6 ближайших соседей центральной точки подсвечены, пунктир к каждому.
  - Счётчик «соседей: **6**».

**Плотность упаковки.** Под сценой, маленький справочный текст:
«плотность упаковки: **78.5%**» для $\mathbb Z[i]$, «плотность
упаковки: **90.7%**» для $\mathbb Z[\omega]$. При переключении число
меняется.

### Подпись (одна строка)

Меняется по режиму:

- `Zi`: *$\mathbb Z[i]$ — 4 соседа, плотность 78,5%. Есть пустоты.*
- `Zomega`: *$\mathbb Z[\omega]$ — 6 соседей, плотность 90,7%.
  Пустот нет.*

## Backstage

Разделы:

1. **$\omega$ как точка.** Короткое напоминание: $\omega = e^{2\pi i/3}
   = -\tfrac{1}{2} + \tfrac{\sqrt{3}}{2} i$ — первообразный корень
   степени 3 из единицы. $1 + \omega + \omega^2 = 0$.
2. **$\mathbb Z[\omega]$ как кольцо.** Эйзенштейновы целые — это
   $\{ a + b\omega : a, b \in \mathbb Z \}$. Это **кольцо с единственным
   разложением на простые** (принцип главных идеалов). В нём есть
   нормальные простые (например, 2 и 5), и **есть расщепляющиеся**
   (7 = $(3 + \omega)(3 + \bar\omega)$) — именно это лежит в основе
   GQ128 и иерархии слайда 13.
3. **Теорема 1910 года Туэ.** Плотнейшая упаковка кругов на плоскости —
   гексагональная, плотность $\pi/\sqrt{12} \approx 0.9069$. Это
   первая из пяти известных оптимальностей упаковок шаров
   (1D, 2D, 3D, 8D, 24D).
4. **Eisenstein 1844.** Оригинальная работа о кубической взаимности.
   Эйзенштейн ввёл кольцо $\mathbb Z[\omega]$ ровно для того, чтобы
   обобщить закон взаимности Гаусса. Это был алгебраический прорыв;
   геометрическая плотность упаковки оказалась **побочным продуктом**.
5. **Биологический мост.** Авторский вклад:
   - В **fovea** человеческого глаза колбочки укладываются в
     гексагональную мозаику с шагом ~2–3 мкм (диаметр центра колбочки
     ≈ 350 нм). Это **упирается в дифракционный предел Аббе** для
     видимого света.
   - Ганглиозные клетки вычисляют center-surround разницу — это
     буквально **аппаратный wavelet**.
   - Природа, видимо, нашла $A_2$ как оптимальную решётку задолго до
     Thue. Подробнее — `raw-ai-articles-dirty/hexagonal wavelets paper.pdf`
     (используется как **источник фактов**, не как цитата).
6. **Взгляд вперёд.** Слайд 11 вводит 7 цифр в базе $\beta = 2 -
   \omega$, и теперь становится ясно, **почему именно 7**: это «сам
   шар + 6 соседей». На слайде 13 покажется, что это универсальный
   принцип — $p = \kappa + 1$ для всех решётых размерностей.

**Математика.**

```ts
// Гауссовы узлы: [i, j] → (i, j).
function gaussianNodes(R: number) {
  const out: [number, number][] = [];
  for (let i = -R; i <= R; i++)
    for (let j = -R; j <= R; j++) out.push([i, j]);
  return out;
}

// Эйзенштейновы узлы: [a, b] → (a + b·Re(ω), b·Im(ω)).
const OMEGA_RE = -0.5;
const OMEGA_IM = Math.sqrt(3) / 2;

function eisensteinNodes(R: number) {
  const out: [number, number][] = [];
  for (let a = -R; a <= R; a++)
    for (let b = -R; b <= R; b++) {
      out.push([a + b * OMEGA_RE, b * OMEGA_IM]);
    }
  return out;
}

// Радиус упаковки — половина ближайшего расстояния.
// Gaussian: 0.5. Eisenstein: 0.5 (т.к. ближайшие соседи на единичном
// расстоянии в обоих случаях при такой нормировке).
```

**Анимация переключения.**

- Все узлы и круги — через `framer-motion` `layoutId`, переезжают на
  свои новые позиции за 600 мс spring. Это ключевой визуальный эффект:
  квадрат «прогибается» в гексагон.
- Подсветка соседей и пунктиры — cross-fade (300 мс).
- Счётчик «4 → 6» — цифра анимируется tik-tak (быстрый пролёт через
  промежуточные цифры 5), 350 мс. Это подчёркивает, что добавляется
  **один конкретный сосед**.
- Плотность в процентах — аналогично, цифры «78.5% → 90.7%».

**Reader mode.**

- Сцена на всю ширину.
- Счётчик соседей уходит в заголовок сцены, цифра крупная.

**Reduced motion.**

- Переключение мгновенное, без layout-анимации, без тик-так.

**A11y.**

- Переключатель — `role="radiogroup"`.
- SVG — `role="img"` + `aria-label` с описанием текущей решётки.
- Счётчик соседей — `aria-live="polite"`.

## Проверка «кончик скотча» и «слайд с середины»

- **Кончик скотча:** картина квадратной решётки с подсвеченными 4
  соседями — образ из учебника.
- **Слайд с середины:** в гексагональном режиме 6 соседей светятся,
  плотность 90,7% — **самодостаточная** картинка.

## Что НЕ делать

- Не писать формулы $\omega$, расщепление $7 = (3+\omega)(3+\bar\omega)$
  на сцене. Всё в бэкстейдж.
- Не добавлять третий режим (например, $\mathbb Z$, одномерный) —
  это путает.
- Не рисовать оси $x, y$. Сетка узлов сама говорит про координаты.
- Не показывать «сколько пустоты остаётся» в $\mathbb Z[i]$
  отдельной графикой. Зритель видит пустоты напрямую — круги не
  касаются по диагонали.
