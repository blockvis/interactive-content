# Slide 05 — Декарт: сетка на плоскости

> Читать сначала: [`_shared.md`](./_shared.md), затем `../BRIEF.ru.md` §7 (строка слайда 5).

## Роль в арке доклада

Завершение исторического блока 3 → 4 → 5. Пифагорейцы отказались от чисел
(слайд 3), Евдокс построил геометрию без чисел (слайд 4). Декарт 1637
года **накладывает сетку на плоскость** — и числа возвращаются в
геометрию через заднюю дверь, как пара координат.

Слайд задаёт словарь, который сразу понадобится на слайде 6
(комплексные числа = точка плоскости = пара чисел).

## Длительность и «кончик скотча»

- На сцене: 45–60 секунд.
- **Кончик скотча:** пустая плоскость и движущаяся по ней точка. Зритель
  **мгновенно** понимает, что эту точку как-то надо назвать. Нажатие
  **«наложить сетку»** делает это названием.

## Content

### Визуал

Центр сцены — область ~700×500 px с **одной свободно плавающей точкой**,
которую зритель может таскать мышкой (или пальцем). Больше ничего.
Снизу — одна кнопка-переключатель «Наложить сетку».

При нажатии на кнопку:

1. Появляется координатная сетка (квадратная, шаг 1 единица).
2. Появляются оси $X$ и $Y$ с подписями.
3. Рядом с точкой проявляется её адрес — пара $(x, y)$, обновляющаяся
   в реальном времени при перетаскивании.

После этого перетаскивание точки показывает, как меняются обе
координаты — это **и есть** демонстрация Декартова воссоединения.

Повторное нажатие кнопки убирает сетку и адрес, оставляя голую точку.

## Backstage

Разделы:

1. **«Géométrie» Декарта, 1637.** Короткая контекстная заметка:
   приложение к «Рассуждению о методе». Декарт показал, что
   **алгебраические уравнения = кривые на плоскости**. Это и есть
   воссоединение — выстраданное греками двумя тысячелетиями геометрии
   без чисел.
2. **Предшественники.** Oresme (XIV век): графики функций.
   Ферма (тоже 1630-е): независимое открытие координат.
3. **Мост на следующий слайд.** Если точка теперь — пара $(x, y)$, то
   $(x, y)$ можно **перемножить как единое целое**. Это и есть
   комплексные числа. Слайд 6 покажет, что «перемножение точек» — не
   просто «перемножение двух чисел отдельно», а **содержательная
   операция**, у которой есть геометрический смысл.

## Компоненты для реализации

### `CartesianGrid`

**Файл.** `presentations/gq128-beauty-presentation/components/CartesianGrid.tsx`.

**Директива.** `:::cartesian-grid`.

**Props.**

```ts
type CartesianGridProps = {
  // Начальные координаты точки. Default: [1.8, 1.2].
  initialX?: number;
  initialY?: number;
  // Диапазон видимой области по каждой оси. Default: 4
  // (то есть видно от -4 до +4).
  range?: number;
};
```

**Визуал.**

- SVG-область с `viewBox="-range -range 2*range 2*range"`.
- Точка — круг радиуса 8 px (в экранных координатах), fill
  `var(--class-accent)`, stroke `var(--class-primary)`.
- При наведении / перетаскивании — радиус 10 px.
- Курсор — `grab` в покое, `grabbing` во время drag.
- Сетка (когда активна): тонкие линии, `stroke: var(--class-muted)`,
  0.5 px. Оси — чуть толще (1 px) и другого цвета `var(--class-primary)`.
- Подписи осей ($X$, $Y$, цифры делений) — мелкие, 12 px. Цифры — на
  целочисленных отметках, не на всех.
- Адрес точки — отдельный бейдж рядом с точкой (следует за ней, но с
  лёгким offset), рендерится как `<foreignObject>` или абсолютный div.
  Формат: `(x, y)` с двумя цифрами после запятой.

**State.**

```ts
const [showGrid, setShowGrid] = useState(false);
const [pos, setPos] = useState<[number, number]>([initialX, initialY]);
```

**Drag.**

- `onPointerDown` на точке → `setCapture`.
- `onPointerMove` → пересчёт координат из screen в SVG-coords.
- `onPointerUp` → `releaseCapture`.
- Ограничение: точка не выходит за прямоугольник `[-range, range]^2`.

**Анимация.**

- Переключение сетки: `framer-motion`, fade-in 300 мс с stagger по
  линиям (сначала оси, потом сетка, потом подписи).
- Адрес-бейдж появляется после сетки (задержка 250 мс).
- Без сетки обратно: всё скрывается за 200 мс.

**Reader mode.**

- SVG ужимается до ширины контейнера, но минимум 280 px.
- Кнопка «Наложить сетку» — под SVG, во всю ширину.
- Бейдж с адресом не перекрывает точку (приоритет — точка видна, бейдж
  уходит ниже/выше при коллизии).

**Reduced motion.**

- Сетка появляется без анимации (мгновенно).

**A11y.**

- Точка — `role="slider"` с `aria-valuenow-x`, `aria-valuenow-y`
  (нестандартно, но понятно), плюс `aria-label="Точка на плоскости,
  x равен 1.8, y равен 1.2"` (обновляется).
- Клавиатура: стрелки сдвигают точку на 0.1 единицы, Shift+стрелки — на
  1.0 единицы.
- Кнопка «Наложить сетку» — стандартный `<button>`, `aria-pressed`.

## Токены и стили

- Оси: `var(--class-primary)`.
- Сетка: `var(--class-muted)`.

  var(--class-primary); color: var(--class-foreground)`. Моно-шрифт
  для `(x, y)`.

## Проверка «кончик скотча» и «слайд с середины»

- **Кончик скотча:** пустая плоскость и точка — зритель уже интуитивно
  знает, что её нужно как-то назвать.
- **Слайд с середины:** если зайти с активной сеткой, видно точку с
  бейджем `(1.8, 1.2)` на сетке. Это **самодостаточная картинка** даже
  без предварительного клика.

## Что НЕ делать

- Не рисовать на сцене стрелочки «вот это x, вот это y, а вот пара
  (x, y)». Весь слайд держится на одном жесте «наложить сетку».
- Не добавлять вторую точку / вектор / умножение. Это всё — сюжет
  следующего слайда (комплексные числа).
- Не показывать алгебраические формулы Декарта (полиномы и кривые).
  Исторически это часть его работы, но к сюжету доклада не ведёт.
