# Slide 09 — Позиционные системы с комплексным основанием

> Читать сначала: [`_shared.md`](./_shared.md), затем `../BRIEF.ru.md` §7 (строка слайда 9).

## Роль в арке доклада

Переходный слайд перед долгом №3. На слайдах 3–8 мы говорили про
**геометрию комплексных чисел**; теперь впервые — про **арифметику**.
Ключевой вопрос: а что, если основание системы счисления — **само
комплексное число**?

Зритель увидит, что это не диковинка, а естественное обобщение
двоичной и троичной системы, и что на комплексной плоскости есть
особые базы, которые дают **красивые области сходимости** (фракталы,
мостящие плоскость).

Этот слайд подготавливает слайд 11 (7 цифр Госпера) — там появится
конкретная база $\beta = 2 - \omega$, а здесь зритель учится **смотреть
на базу**.

## Длительность и «кончик скотча»

- На сцене: 75 секунд (немного дольше: здесь новый объект).
- **Кончик скотча:** ползунок комплексного числа $\beta$ на плоскости,
  и **сразу** видно область сходимости. Это фрактальный образ. Эффект
  «ого, что это» действует мгновенно, без объяснения.

## Content

### Визуал

Сцена — одна интерактивная комплексная плоскость (SVG/canvas
~640×500 px в presentation mode).

- Свободная ручка **$\beta$** — точка на плоскости, которую можно
  таскать. По умолчанию стоит на $\beta = 2i$ (классический пример
  Кнута).
- Вокруг точки $\beta$ — небольшая подсветка, подпись «$\beta$».
- **Область сходимости** $K_\beta$ — **единичный интервал** в системе
  счисления с основанием $\beta$: множество всех чисел $z = \sum_{k \ge 1}
  d_k \beta^{-k}$ для всех возможных последовательностей цифр $d_k \in
  D$ (алфавит цифр). Это фрактал — IFS с двумя-шестью отображениями в
  зависимости от $|D|$.
- Область закрашивается заливкой `var(--class-accent)` с
  `opacity: 0.35`, контур — `var(--class-primary)` 1 px.
- **При перетаскивании** $\beta$ область мгновенно (или с
  микро-задержкой рендера) пересчитывается и перерисовывается.

Над сценой — селектор алфавита цифр:

- «Двоичный» (0, 1) — классический случай для $\beta = 2i$.
- «Троичный» (0, 1, 2) — для $\beta = 3i$, $\beta = 2 + i$ и подобных.
- «Семеричный» (0…6) — **намёк** на то, что будет на слайде 11.

По умолчанию — семеричный, чтобы слайд заранее подсвечивал связь со
слайдом 11 (Госпер).

**Пресеты-подсказки.** Три клика:

- «$\beta = 2i$» — ставит ручку в $2i$, алфавит 0,1. Показывает
  классический фрактал Кнута.
- «$\beta = -1 + i$» — алфавит 0,1. База Penney, 1965.
- «$\beta = 2 - \omega$» — алфавит 0…6, семеричный Госпер. База
  со слайда 11.

При смене пресета ручка едет на своё новое место (200 мс), область
перерисовывается.

## Backstage

Разделы:

1. **Как считать область сходимости.** Если $D = \{d_0, \ldots, d_{p-1}\}$
   — алфавит, а $\beta$ — основание, то единичный интервал
   $K_\beta = \{ \sum_k d_{n_k} \beta^{-k} \}$ — это аттрактор IFS из
   $p$ сжимающих отображений $f_d(z) = (z + d) / \beta$, при условии
   $|\beta| > 1$. Это частный случай самоаффинного фрактала.
   Можно привести 3–4 строки KaTeX, но **без** полных доказательств.
2. **Knuth 1960.** «Imaginary radix representations»: база $\beta =
   2i$ с цифрами 0,1,2,3 — единственная позиционная система, в которой
   **нет знака** (положительные и отрицательные числа представимы без
   минуса). Классическая работа.
3. **Penney 1965.** «A binary system for complex numbers» — база $\beta
   = -1 + i$ с цифрами 0, 1. Любое целое гауссово число имеет
   **однозначное** представление.
4. **Взгляд вперёд (слайд 11).** $\beta = 2 - \omega$ — база в
   $\mathbb Z[\omega]$. $|\beta|^2 = 4 - 2 + 1 = \ldots$
   (в $\mathbb Z[\omega]$ форма $|a + b\omega|^2 = a^2 - ab + b^2$;
   для $a = 2, b = -1$: $4 + 2 + 1 = 7$). Значит, единичный интервал
   этой базы — знаменитый **остров Госпера**, вмещающий 7 цифр, и он
   будет кульминацией слайдов 11–12.
5. **Почему «долг».** Стандартное учебное изложение «позиционных систем»
   жёстко привязано к $\beta \in \mathbb N_{\ge 2}$, хотя всё обобщается
   на $\mathbb C$ совершенно естественно, и с **геометрической** точки
   зрения это красивее. Это и есть четвёртый, «мягкий» технический
   долг — за который GQ128 отвечает.

## Компоненты для реализации

### `ComplexBaseExplorer`

**Файл.** `presentations/gq128-beauty-presentation/components/ComplexBaseExplorer.tsx`.

**Директива.** `:::complex-base-explorer`.

**Props.**

```ts
type ComplexBaseExplorerProps = {
  // Начальная база. Default: [2, -1] (т.е. 2 - ω в
  // ω-представлении, при selectedDigits = "seven"). Но в
  // декартовых координатах она равна 2 - (-0.5, √3/2) = (2.5, -√3/2).
  // Хранится и подаётся как [re, im] по умолчанию.
  initialBeta?: [number, number];
  // Начальный алфавит. Default: "seven".
  initialDigits?: "binary" | "ternary" | "seven";
  // Разрешение рендера фрактала. Default: 120 (количество итераций).
  fractalIterations?: number;
  // Количество сэмплов в рендере. Default: 40000.
  samples?: number;
};
```

**Визуал и математика.**

**Рендер области сходимости — через chaos game.**

- Отобразить алфавит: `binary → [0, 1]`, `ternary → [0, 1, 2]`,
  `seven → [0, 1, 2, 3, 4, 5, 6]`. Можно расширить интерпретацию
  для `seven` на сбалансированный набор $\{0, \pm 1, \pm \omega, \pm
  \omega^{-1}\}$ (Doppler-EGA), но это **опционально** — оставим на
  первой итерации стандартный $\{0, \ldots, 6\}$ (BRIEF: педагогический
  переход слайд 11 → 14).
- Рендер через **обратное IFS**: взять стартовую точку `z = (0, 0)`,
  далее `samples` раз выбирать случайно цифру $d$ и считать `z =
  (z + d) / beta`. Каждую посещённую точку рисовать.
- Точки рендерятся в **canvas** (SVG не потянет 40 тыс. точек). Canvas
  обёрнут в `<foreignObject>` или подложен под SVG-слой с ручкой.
- Цвет точки — `var(--class-accent)` с `alpha = 0.25`, накапливается
  в плотность.

**Ручка $\beta$.**

- Отдельный SVG-слой поверх canvas.
- Большая точка, stroke `var(--class-primary)`, fill белый,
  перетаскиваемая.
- Координаты $\beta$ отображаются рядом: `β = re.re + re.im·i` (если
  алфавит binary/ternary) или `β = a + b·ω` (если seven). Формат
  выбирается по алфавиту.

**Пресеты.**

Три кнопки в строке над сценой. Клик — плавный переезд ручки (200 мс)
+ пересчёт рендера.

**State.**

```ts
const [beta, setBeta] = useState<[number, number]>(initialBeta);
const [digits, setDigits] = useState(initialDigits);
// Фрактал пересчитывается реактивно через useEffect при изменении
// beta или digits — с debounce 80 мс, чтобы не тормозить drag.
```

**Оптимизация.**

- Перед перетаскиванием — **пре-рендер низкого разрешения** (5000
  сэмплов, 60 итераций).
- После отпускания drag — **полный рендер** (40000, 120).
- Debounce в 80 мс на `beta` изменения, чтобы не пересчитывать
  10 раз в секунду.

**Анимация.**

- Плавность обеспечивается через debounce + двойной рендер
  (low-res → full-res).
- Переезд ручки по клику на пресет — `framer-motion` spring.
- При смене алфавита — cross-fade canvas (200 мс).

**Reader mode.**

- Canvas занимает всю ширину, высота пропорциональна.
- Селектор алфавита — уходит под сцену.
- Пресеты — тоже под сцену, в строку или сетку.

**Reduced motion.**

- Выключить spring-анимацию переезда ручки.

**A11y.**

- Ручка — `role="slider"` с двумя значениями (re, im).
- Стрелки клавиатуры ←↑→↓ — шаг 0.05 единицы.
- Селектор алфавита — `role="radiogroup"`.
- Canvas — `role="img"` + `aria-label` с описанием: «Область сходимости
  системы счисления с основанием $\beta$ и $p$ цифрами. Текущая $\beta$
  и её приближённая форма.»

**Производительность.**

Если 40 тыс. сэмплов для полного рендера на слабом железе не тянутся
за 100 мс — уменьшить до 20 тыс. и увеличить alpha для компенсации.
Подбирается при отладке.

## Проверка «кончик скотча» и «слайд с середины»

- **Кончик скотча:** фрактальная область — визуальный магнит. Даже
  без подписи работает.
- **Слайд с середины:** любой момент — картина самодостаточна. Если
  зритель зашёл, когда стоит пресет «$\beta = 2 - \omega$», видит
  **силуэт острова Госпера** уже до слайда 12.

## Что НЕ делать

- Не показывать на сцене формулу IFS и выкладки. Они — в бэкстейдже.
- Не показывать два фрактала одновременно для сравнения. Один
  ползунок — одна картинка.
- Не использовать пресет «$\beta = 2 - \omega$» **до слайда 11**,
  если сценарий требует держать Госпера в тайне до кульминации. Но
  здесь это **намёк**, не раскрытие — картинку зритель видит, но без
  имени. Это осознанный ход, автор подтверждает.
