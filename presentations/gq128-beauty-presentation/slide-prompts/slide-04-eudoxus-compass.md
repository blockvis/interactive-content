# Slide 04 — Евдокс: перемножение отрезков циркулем

> Читать сначала: [`_shared.md`](./_shared.md), затем `../BRIEF.ru.md` §7 (строка слайда 4).

## Роль в арке доклада

Мостик между катастрофой (слайд 3) и воссоединением (слайд 5).
Пифагорейцы, не сумев поместить $\sqrt 2$ в числа, **отказались от
чисел**. Евдокс и вся V книга «Начал» — ответ геометрии: «мы не знаем,
что такое число, но знаем, что такое пропорция отрезков». На этом
фундаменте греческая математика прожила две тысячи лет, пока Декарт не
склеил арифметику и геометрию обратно.

Самый тонкий слайд исторического блока: здесь легко соскользнуть в
лекцию по теории пропорций. Держимся одного жеста.

## Длительность и «кончик скотча»

- На сцене: 60–75 секунд.
- **Кончик скотча:** даны три отрезка — единичный эталон, отрезок «$a$»,
  отрезок «$b$». Сцена буквально **спрашивает**: как из этого построить
  отрезок длины $a \cdot b$? Циркуль и линейка лежат рядом. Зритель
  задумывается **до того**, как докладчик скажет хоть слово.

## Content

### Визуал

Центр сцены — простая **геометрическая доска** (область ~600×400 px в
presentation mode). На доске:

- Горизонтальная опорная прямая.
- Три горизонтальных отрезка слева, подписанные `1`, `a`, `b` (разной
  длины, но тот, что подписан `1`, — **единичный эталон** и явно
  выделен: другой цвет и микро-линеечка-метка).
- Справа — пустое место для результата.

Над доской — две кнопки-инструмента: «Линейка» и «Циркуль». Кликнешь
на линейку — появляется полупрозрачная линейка, которая следует за
курсором и позволяет построить прямую через две точки. Кликнешь на
циркуль — появляется циркуль, с которым можно снять расстояние между
двумя точками и перенести его на другую прямую.

По умолчанию стоит **кнопка-подсказка «Подсказать шаги»**, которая
делает каноническое построение автоматически, по одному шагу, с
паузой 0.8 с между шагами:

1. Провести прямую из одной точки (назовём её $O$).
2. От $O$ по первой прямой отложить единичный эталон $[O, E]$.
3. От $O$ по другой прямой отложить $a$ → точка $A$.
4. От $E$ отложить $b$ вдоль первой прямой → точка $B$.
5. Провести прямую через $E$ и $A$.
6. Провести прямую через $B$, параллельно $EA$, — пересекает вторую
   прямую в точке $C$.
7. Отрезок $[O, C]$ — это $a \cdot b$.

Каждый шаг сопровождается микро-подсказкой снизу (сменяющаяся строка).

После завершения построения — короткая анимация «снятия циркулем»
длины $[O, C]$ и переноса её на результирующую область справа, где
теперь стоит подписанный отрезок длины $ab$.

### Подпись (одна строка)

> *Без единого числа — через подобие треугольников.*

## Backstage

Разделы:

1. **Конструкция по Евклиду (VI.12).** Формальное изложение: «даны
   три отрезка, построить четвёртый пропорциональный». Именно это
   построение делает интерактив.
2. **Теория пропорций Евдокса (V книга «Начал»).** Зачем это было
   нужно: без неё грекам пришлось бы остановиться на целых числах.
3. **Что сломалось.** Построение работает только для конечного
   отрезка. Понятие «число» как объект арифметики отделилось от
   «длины» как объекта геометрии на две тысячи лет.
4. **Короткий намёк:** Декарт (слайд 5) **одним жестом** вернёт числа
   обратно, наложив на плоскость сетку.

## Компоненты для реализации

### `EudoxusCompass`

**Файл.** `presentations/gq128-beauty-presentation/components/EudoxusCompass.tsx`.

**Директива.** `:::eudoxus-compass`.

**Props.**

```ts
type EudoxusCompassProps = {
  // Длины трёх исходных отрезков в условных единицах SVG.
  // Default: a=2.0, b=2.6. Единица — 1.0 по определению.
  a?: number;
  b?: number;
  // Скорость автопостроения, мс между шагами. Default: 800.
  stepMs?: number;
};
```

**Визуал.**

- SVG-область, координаты в условных единицах.
- Точки — маленькие кружки, подписанные заглавными латинскими буквами
  (`O`, `E`, `A`, `B`, `C`).
- Линии чертежа — `stroke: var(--class-primary)`, 1.5 px.
- Вспомогательные прямые (те, что появляются в процессе) — чуть светлее
  и пунктирные: `stroke-dasharray: 4 3; stroke: var(--class-primary);
  opacity: 0.5`.
- Результирующий отрезок $[O, C]$ после шага 7 — жирный,
  `stroke: var(--class-accent)`, 3 px.
- «Линейка» и «Циркуль» — кастомные SVG-иконки (можно простые: линейка
  как прямоугольник с засечками, циркуль как буква «Λ»).
- Подсветка текущего шага — мягкая пульсация активной фигуры (если не
  reduced-motion).

**State.**

```ts
const [mode, setMode] = useState<"idle" | "ruler" | "compass">("idle");
const [step, setStep] = useState(0); // 0..7, 0 = исходное
const [autoPlaying, setAutoPlaying] = useState(false);
```

**Анимация.**

- Автопостроение через `setInterval(stepMs)`.
- Каждый шаг — новая SVG-линия или точка с лёгким fade-in (180 мс).
- При достижении step=7: пауза 600 мс, потом перенос $[O, C]$ →
  результирующая область: анимация `path` по дуге, с «циркульной»
  траекторией. 600 мс.
- Кнопка «Назад к началу» сбрасывает `step` в 0 с обратным fade-out.

**Ручная работа** (mode = "ruler" / "compass"). В первой редакции —
**упрощённая ручная работа**:

- `ruler`: клик на две существующие точки → появляется линия через них.
- `compass`: клик на точку-центр, затем клик на другую точку (радиус) →
  появляется окружность.

Ручная работа — **не обязательная фича**. Если упрощает жизнь
реализации, её можно вынести в бэкстейдж как отдельную песочницу и на
сцене оставить только кнопку «Подсказать шаги». Это подтверждается
автором.

**Reader mode.**

- SVG заполняет ширину.
- Кнопки-инструменты уходят под SVG в строку.
- Шаговые подсказки остаются под SVG, не поверх.

**Reduced motion.**

- Автопостроение идёт без пульсаций и без анимации переноса циркулем
  в конце. Показываются все фигуры разом после нажатия «Подсказать
  шаги».

**A11y.**

- Кнопки инструментов — `<button>`.
- Контейнер SVG — `role="img"` + `aria-label` с текстовым описанием
  построения.
- Текущая подсказка-шаг — в `aria-live="polite"` регионе.

## Проверка «кончик скотча» и «слайд с середины»

- **Кончик скотча:** три отрезка и два инструмента — картинка
  **задаёт задачу** до всякой лекции.
- **Слайд с середины:** если зайти после шага 4-5, видно частично
  построенное подобие треугольников. Подпись остаётся одной и той же
  и не теряет смысла.

## Что НЕ делать

- Не писать на сцене формулу пропорции
  $\frac{1}{a} = \frac{b}{ab}$ — это уже будет лекция. Она в
  бэкстейдже.
- Не анимировать «умножение чисел» (типа `2 × 3 = 6` в углу). Этот
  слайд весь про **геометрию без чисел**.
- Не ставить на сцене циркуль с углами и подсветками углов — это
  добавляет абстракции, которая здесь мешает.
